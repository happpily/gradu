# GATNet算法优势增量分析

> 问题：我的算法相比标准方法实现了什么样的优势？不是简单把GAT套用到无线场景，而是做了哪些针对性设计？

---

## 总览

GATNet并非标准GAT的直接应用，而是在**注意力计算、特征保留、输出约束、损失函数、训练策略**五个层面针对无线功率分配问题做了7项定制化设计。

---

## 1. 边特征注入的注意力计算

**普通做法**：标准GAT的注意力仅基于节点特征
```
α_{ij} = softmax(LeakyReLU(a^T [Wh_i || Wh_j]))
```
问题：在无线干扰图中，干扰链路j对目标链路i的干扰强度由信道增益h_{j,i}直接决定。如果注意力不看边特征，模型需要从节点特征中"间接推测"干扰强度，效率低且不精确。

**GATNet的做法**：打开PyG GATConv的edge_dim参数
```python
GATConv(in_channels=hidden_dim, out_channels=D, heads=H, edge_dim=2, ...)
```
使注意力公式扩展为：
```
α_{ij} = softmax(LeakyReLU(a^T [Wh_i || Wh_j || W_e · e_{j,i}]))
```
其中边特征 e_{j,i} = [log(1+g_{j,i}), log(1+g_{i,j})] 显式编码了双向干扰信道增益。

**效果**：强干扰边天然获得高注意力权重，弱干扰边天然获得低权重。模型不需要"猜测"干扰强度——信道增益直接告诉模型该关注谁。

---

## 2. 无自环设计

**普通做法**：标准GAT默认加自环（add_self_loops=True），因为节点自身的特征参与消息聚合是有意义的（如社交网络中人的特征也会影响自己的分类）。

**GATNet的做法**：
```python
GATConv(..., add_self_loops=False)
```
**原因**：在干扰图模型中，"邻居"的定义就是"干扰源"。链路不会干扰自己，所以不需要自环。直传信道h_{ii}的信息已经编码在节点特征中，通过第3点所述的跳跃连接直送输出层，不依赖GAT层传递。

---

## 3. 双跳跃连接 —— 核心设计亮点

**第一层：层间残差连接**
```python
h = ELU(LayerNorm(h + GAT(h)))  # 每层都加
```
作用：防止4层GAT堆叠导致的梯度消失和过度平滑。这是常见做法但不是每个模型都用。

**第二层：输入跳跃连接** —— 这是你的关键创新
```python
h_out = torch.cat([h_final, x_init], dim=-1)  # [128维GAT输出 + 3维原始特征]
```
这个设计针对功率分配问题有明确的物理动机：经过4层GAT邻居聚合后，节点特征 h_final 已经混合了大量来自邻居（干扰源）的信息，链路自身的直传信道增益 g_{ii} 的精确数值可能被"稀释"。但功率分配决策首先依赖的就是"我自己的信道好不好"——如果直传增益信息在GAT层间丢失，模型就不可能做出精确的功率决策。

跳跃连接把原始的三维节点特征 [log(1+g_ii), α_i, log10(σ²)] 直送给输出MLP，保证了直传信道信息有一条"信息高速通道"，不经过任何变换直接到达决策层。

---

## 4. 硬约束输出：Sigmoid × p_max

**普通做法**：用ReLU然后clamp到[0, p_max]，或者用拉格朗日乘子法软约束。
- ReLU + clamp的问题：clamp在边界处梯度为零（神经元坏死）
- 软约束的问题：约束可能不精确满足

**GATNet的做法**：
```python
power = sigmoid(MLP_output) * p_max
```
效果：
- 0 < p_i < p_max 天然严格成立，不需要任何后处理
- Sigmoid在(0,1)内处处可导，梯度永不消失
- 训练结束后直接部署，输出永远合法

---

## 5. 五合一损失函数

**普通做法**：直接用 -sum_rate 作为损失。

**GATNet的做法**：
```
Loss = 1/(sum_rate + ε)                    ← 倒数目标（速率越高损失越小越稳定）
     + λ_power × mean(power)               ← 鼓励节能
     + λ_fair  × (1 - Jain_fairness)       ← 避免少数链路垄断资源
     + λ_qos   × mean(relu²(rate_min − R)) ← QoS门限，L2惩罚（大违反重罚）
     + λ_sinr  × mean(relu²(SINR_min − γ)) ← SINR保底，L2惩罚
```

四个设计细节：

**（1）倒数目标而非负速率**：`1/(R+ε)`优于`-R`的原因是——当总速率较小时（训练初期），`1/R`的梯度很大，推动模型快速提升速率；当速率接近收敛时，`1/R`的梯度逐渐缩小，允许模型精细优化而不震荡。这相当于一种自适应的学习率调节。

**（2）L2平方惩罚**：`relu(threshold - value)²` 而非 `relu(threshold - value)`。L2惩罚对大违反的惩罚力度远大于L1——一条链路严重不满足QoS会受到比两条链路轻微不满足更大的惩罚，促使模型"先救最差的"。

**（3）独立可调的λ权重**：config.py 中每项λ独立设置，可以根据业务场景灵活调整：
```
lambda_power    = 0.01   # 轻度节能约束
lambda_fairness = 0.05   # 中度公平性约束
lambda_qos      = 0.15   # 较重QoS约束（保障最低通信质量优先）
lambda_sinr     = 0.20   # 最重SINR约束（物理层可靠性）
```

**（4）五项从四个维度表征通信系统需求**：频谱效率 + 能量效率 + 用户公平性 + 服务质量。相比之下，文献[4]仅使用 -sum_rate 单目标。

## 6. 在线拓扑采样训练 —— 一种免费的泛化增强

**普通做法**：固定训练集，反复迭代训练。

**GATNet的做法**：
```python
for epoch in range(3000):
    for _ in range(16):
        data, _ = generate_network_data(...)  # 每次随机生成新拓扑
        # 随机种子seed=None，每次的链路位置、信道实现、干扰分布都不同
    # 在这个批次上训练
```

训练总量：3000轮 × 16样本/轮 = **约48,000个不同的网络实例**。

**为什么这很重要**：无线功率分配是一个"策略学习"问题而非"模式识别"问题。模型需要学到的是"如何根据信道条件分配功率"这个通用策略，而不是"在特定拓扑下的最优功率分配"。在线采样确保了模型在训练过程中见过足够多样化的拓扑结构，天然防止了对特定拓扑配置的过拟合。

这一策略的有效性在各章的实验中得到了验证——模型在不同网络规模和干扰强度下都表现稳定。

## 7. WMMSE功率下限 + GATNet-WMMSE混合

**普通WMMSE的问题**：标准WMMSE迭代过程中可能将弱链路（直传增益极低）的功率逐步压到零，导致该链路的SINR→0、SINR_dB→-∞。即使只有一条链路出现这种情况，平均SINR (dB)就会被拉到极低值。

**GATNet的做法**（在baselines.py中）：
```python
p_min_frac = 0.02  # 每条链路至少分配2%峰值功率
p = clip(p_new, 0.02*p_max, p_max)
```
2%的功率下限对和速率的损失微不足道，但有效避免了SINR指标因为一条零功率链路而全面失真的问题。这是一个工程实践中的关键细节。

**GATNet+WMMSE混合方案**：用GATNet的7ms推理输出作为WMMSE的起点，仅需8次迭代就收敛到全迭代（60次）性能的97.2%。这不是一个简单的"堆叠"，而是利用了GATNet学到的"高质量邻域"——初始解已经落在WMMSE收敛点附近，大幅缩短了迭代路径。

## 总结对照表

| 设计层面 | 普通做法 | GATNet的做法 | 增量优势 |
|---|---|---|---|
| 注意力输入 | 仅节点特征 | 节点+边特征联合计算 | 干扰强度显式影响注意力 |
| 自环 | 默认加自环 | 不加自环 | 避免冗余，干扰图语义更清晰 |
| 特征保留 | 仅层间残差或无 | 层间残差+输入跳跃连接 | 直传信道信息直送到输出层 |
| 输出约束 | ReLU+clamp或软约束 | Sigmoid×p_max | 梯度永不断，约束天然满足 |
| 损失函数 | -sum_rate单目标 | 1/R + 4项正则 | 频谱效率+能效+公平+QoS+可靠性 |
| 训练数据 | 固定训练集 | 每轮随机采样新拓扑 | 48,000种拓扑的泛化经验 |
| WMMSE实现 | 允许零功率 | p_min=2%功率下限 | SINR指标不因极值失真 |
| 混合方案 | 无 | GATNet初值+8次WMMSE精调 | 97.2%性能 / 13.3%迭代量 |

---

## 一句话总结

在注意力层面：用边特征让模型"看到"干扰强度。
在特征层面：用跳跃连接保护直传增益不被稀释。
在输出层面：用Sigmoid保证功率永远合法。
在训练层面：用五项损失+在线采样实现多维优化和强泛化。
在工程层面：用功率下限+混合精调兼顾理论合理性和实际可用性。
